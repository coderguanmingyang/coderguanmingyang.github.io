About Git

by Guan Mingyang

2017/9/8 12:55:41

Introduction

当开发一个大型项目时，尤其是多人开发时，代码的同步和管理显得尤为重要。笔者在进行实际项目开发时，虽然是一个人，但是还会面临着版本控制等问题。当前比较流行的代码仓库有GitHub、gitlab和bitbucket等，其中主要用git来实现代码管理，本文将对主要常用git命令进行介绍。

Generate ssh-key

首先，你要在本地生成一个ssh密钥，然后将其加入到你的代码仓库中，目的就是让代码仓库识别你的合法身份，能让你对代码仓库进行操作。

  • 首先要查看你的本地有没有生成过ssh密钥，可以通过以下命令（以Linux为例）来查看是否有id_rsa.pub和id_rsa两个文件
    $ cat ~/.ssh/id_rsa.pub
    $ cat ~/.ssh/id_rsa

  • 如果没有以上两个文件的话，就要生成ssh密钥：
    ssh-keygen -t rsa -C "youremail@example.com"
    然后直接一直回车即可，这样就得到了一个密钥。在用户主目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人

  • 将id_rsa.pub中的内容加到你的代码仓库的密钥。
    $ vim ~/.ssh/id_rsa.pub
    点击下图中的add key，将id_rsa.pub中的内容复制进去（此图来源于bitBucket界面）

    Create a new repository

    经过以上的操作，你的本地电脑就可以和远程仓库取得联系了，剩下的事就是要将远程代码仓库和本地仓库进行同步，以便于项目开发。

  • 首先，在代码仓库平台进行新建repository，下图是bitbucket界面（bitbucket允许免费建立private项目），如果在其他平台上有项目也可以直接导入。

  • 然后将新建的仓库进行clone到本地（本地路径就是将来你的本地代码存放位置）
    $ git clone your-repository-address

  • 你新建的项目是空的，你可以直接在本地对应位置下进行代码开发和管理等一系列操作。

    Create and merge a branch

    代码仓库的一个重要功能就是多人分工协作和版本控制，当小A和小B同时在开发一个项目时，那么两个人都会对整个项目做出修改和贡献，如何记录每个人的工作呢？那么就需要引入分支“branch”的概念。
    整个项目是有一条主线master（就是主分支）的，每个人在自己的分支下进行工作，当到一定阶段时，就要将自己的修改commit到master上。代码仓库会记载你的分支的工作，也会记录每个人commit的节点。这样，就能按照commit的不同节点来进行版本回退。

  • 首先，创建一个自己的分支
    $ git checkout -b yourBranchName

以上命令相当于创建分支和切换分支这两条命令的总和
$ git branch yourBranchName
$ git checkout yourBranchName

  • 接下来，你就可以在你的分支下进行工作了，你可以使用以下命令来查看当前分支情况
    $ git branch
    比如说打印如下情况，带星号的表示正处于当前分支
    *guan
    master

  • 那么剩下的就是你在当前分支下进行各种修改了，当你修改一个文件时，想要把修改保存，可以先将其加入本地缓冲区，运用以下命令：
    $ git add yourFile
    当然你也可以用一个命令将所有文件均加入到缓冲区：
    $ git add .

  • 然后，通过commit到自己的分支上
    $ git commit -m "your comments"
    这里注意的是，-m选项指的是添加注释，这个注释将记录到你所做的改动上，即在此之前你的所有git add的文件上。所以，从这点来看，笔者有两个建议，①一定要写注释，方便记录改动②尽量不要使用git add .来将所有文件都加入到缓冲区，而是只将有所修改的文件加入，不然没有修改的文件也会有这个记录。

  • 当你的把自己的工作commit到自己的分支上，想要将自己的分支合并到master上，需要先切换成master分支，然后在合并。
    $ git branch master
    $ git merge --no-ff -m "your comments" yourBranchName
    这里需要强调几点：

    1. merge只能切换到master上才能执行。
    2. 不经过merge操作，你的分支上的改动是不会显示在master上的。
    3. 记得merge时，要加上--no-ff这个选项，其为no fast forward 的缩写，即会记录你分支上的情况。
